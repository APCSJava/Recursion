What's going on?

A solution to fib(10) involves calculating a solution to fib(9) and fib(8). A solution to fib(9) requires a solution to fib(8) and fib(7), etc. If we don't keep track of these calculations but produce them anew, then we are routinely undertaking calculations that have already been performed.

A really good tool for visualizing this process is Jeliot. It is available as a *.jar file that will open up into a Java editor and runtime visualizer. It is worth the download even if you don't use it for full time Java development but occasionally might like a powerful visualization of the call stack.

Another way to trace this code is to drop a breakpoint into the main() method and step your way into/over successive method calls. Try that and watch how the stack grows while executing the code.
