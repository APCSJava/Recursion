Observing inefficient behavior

Notice that we've inserted a print statement that will produce a line of output whenever the method is invoked as well as the argument provided as a parameter.

Run the code and count the lines of output. Consider why so many calls are necessary to calculate fibonacci(10).

You may wish to start with a smaller value than 10 and gradually work back to 10 to see if a pattern emerges.

Alternately, you could modify the class by adding a static counter variable and incrementing it each time fibonacci() is invoked, printing the final value from main().
